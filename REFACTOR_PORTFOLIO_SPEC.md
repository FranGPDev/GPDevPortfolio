# SPEC: Portfolio Profesional Estático (GitHub Pages)
## Objetivo
Crear una Landing Page de alto impacto, minimalista y técnica para atraer reclutadores de Front-end/AEM.

## Stack Tecnológico
- HTML5 Semántico
- CSS3 (Custom Properties / Flexbox / Grid)
- JavaScript (ES6+ Vanila - No frameworks para demostrar dominio de base)
- FontAwesome (Iconos)

## Estructura de Archivos
/ (raíz)
├── index.html          # Estructura principal (Semántica)
├── /css
│   └── styles.css      # Diseño responsivo y animaciones
├── /js
│   └── main.js         # Interactividad (Scroll, filtrado simple)
├── /assets
│   ├── cv_franky.pdf   # Tu currículum en PDF
│   └── /img            # Capturas de tus proyectos MVC
└── README.md           # Explicación técnica del repo

## Secciones Obligatorias
1. **Hero:** Nombre (Mi Franky), Título (Frontend Architect / AEM Developer) y CTA (Call to Action).
2. **Tech Stack:** Grid de iconos con tecnologías (JS, PHP MVC, AEM, Git).
3. **Proyectos:** Cards con link al repo y demo (Incluir el de MangarotaAgent).
4. **Experiencia/Educación:** Timeline técnica.
5. **Contacto:** Links a LinkedIn, GitHub y Email.

## Reglas de Estilo
- Modo Oscuro por defecto (Deep Black #050505 / Neon Cyan #00dbc5).
- Animaciones sutiles de entrada (Intersection Observer).
- Tipografía: Inter o Montserrat (Google Fonts).

## Voz y Tono: 
Todos los comentarios en el código deben estar en primera persona (ej: "Configuro mi header...") y las explicaciones deben ser breves, sinceras y didácticas.
